# init.sql 摘要

- 目标：为 MySQL 8+ 初始化 `logistics` 库，包含表结构与演示数据。
- 步骤：创建数据库（utf8mb4）、按依赖顺序清空旧表，然后重建并写入样例数据。

## 表结构概览
- `sys_user`：用户账号（唯一用户名）、BCrypt 密码、角色/状态、姓名、最近登录、软删除标记。
- `order_record`：订单主表，包含下单日期时间、运单号、机型、SN、备注、分类、状态、金额/币种、重量、客户名、创建人、导入标识、软删除；唯一 SN；索引：订单日期、状态。
- `user_submission`：用户提交的运单，含用户名、运单号、状态（默认 PENDING）、金额、提交日期、备注、软删除；唯一运单号；索引：状态、提交日期。
- `user_submission_log`：用户提交的原文记录（username+内容+创建时间）；索引：用户名、创建时间。
- `hardware_price`：硬件价格表，按日期和品名唯一；包含价格、创建人、软删除；索引：价格日期。
- `settlement_record`：结账记录，引用 `order_record.id`（外键）；含运单号、机型、金额/币种、是否手工、状态、预警、结算批次、应付日期、备注、确认人/时间、软删除；索引：结算批次、状态。
- `sys_log`：操作日志，记录用户名、动作、详情、IP、时间；索引：用户名。

## 种子数据
- 用户：`admin` 与 `operator` 两个账号（BCrypt 密码对应明文 suixiang / operator123），角色分别为 ADMIN、USER。
- 硬件价格：当日 CPU/GPU/主板/内存/硬盘的示例价格多条。
- 日志：管理员登录、导入订单，操作员生成结账的示例记录。
- 注释掉的样例：订单和结账数据的 INSERT 以 `#` 注释保留，可按需启用。
