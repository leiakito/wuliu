# 物流对账前端 (Vue 3 + Element Plus)

本目录包含与 Spring Boot 后端配套的管理端前端，实现登录鉴权、仪表盘、物流单号、结账、用户和日志等模块。

## 技术栈

- Vite + Vue 3 + TypeScript
- Element Plus UI 库
- Vue Router + Pinia
- Axios 封装统一拦截器

## 本地运行

```bash
cd front
npm install
npm run dev
```

默认开发服务器运行在 `http://localhost:5173`，通过 Vite 代理将 `/api` 请求转发到 `http://localhost:8080`。如后端端口不同，请修改 `vite.config.ts` 中的 `proxy.target`。

## 构建

```bash
npm run build
```

构建产物输出到 `dist/`。部署到生产时，将资源托管到任意静态服务器，确保前端与后端同域或配置反向代理。

## 主要功能

- **登录/登出**：基于 Sa-Token，登录成功后在 `Authorization` 头中透传 token。
- **仪表盘**：日期范围筛选，展示订单与结账统计指标。
- **物流单号**：分页筛选、手动新增、Excel 导入、批量抓取、状态变更。
- **结账管理**：生成待结账、分页筛选、金额确认、批量删除（管理员）、导出 Excel。
- **用户管理**：管理员维护账号、角色、状态和密码。
- **操作日志**：管理员分页查看审计日志。
- **硬件价格**：记录每日 CPU/主板/GPU 等硬件价格，管理员可新增/编辑，所有用户都能查询历史价格。

## 默认账号

| 角色     | 用户名  | 密码        |
| -------- | ------- | ----------- |
| 管理员   | `admin` | `admin123`  |
| 用户     | `operator` | `operator123` |

## 角色权限

- 管理员：维护订单、批量抓取/导入、结账确认、导出与用户管理。
- 用户：由管理员创建，仅能通过输入运单号查询当前状态。
- 录入订单时请在“客户账号”中填写客户识别信息（如姓名、公司或渠道备注），方便后续识别。
- 普通用户在“物流单号”页面的查询结果会追加到页面表格中以便留存。

## 环境变量

如需切换后端地址，可在 `.env` 文件中覆盖 Vite 代理：

```
VITE_API_BASE=/api
```

若部署在不同域，可在 `src/api/client.ts` 中更改 `baseURL`。
